중요 수정사항

1. SMI Driver (smi_driver.c)
- ksz8863 chip은 MII management interface와 SMI(Serial management interface) 모두 지원하나 RSTP 기능을 사용하기 위해서는
  SMI, SPI 만 가능한다
- MDC, MDIO 핀을 GPIO로 변경하고 SMI Driver를 구현하였다
- SMI 사용시 SPI CS 핀은 pull-up 상태가 되어야 한다

3. Cyclone TCP stack은 network 기능만 제공하며 FAT File system은 제공하지 않는다 
- MDK4의 FS_CM3.lib를 그대로 사용한다

4. system_lpc43xx.c 사용시 finit() 호출 후 SystemCoreClock이 0이 되는 문제 발생함 

- SystemCoreClock 변수가 data영역(0x20000000)에 할당되는데 이 finit() 호출 후 
  SystemCoreClock이 0이되면서 OS tick에 영향을 준다

- 임시로 finit() 호출 후 SystemCoreClockUpdate()를 호출 하여 값을 복원 시킨다

    void init_card (void) {
        U32 retv;

        finit (NULL);
        #if 1	// finit() 호출 후 SystemCoreClock 값이 0 으로 변경된다.
            SystemCoreClockUpdate();
        #endif
        cmd_dir(NULL);
    }

5. peripheral clocking
- system_lpc43xx.c 사용시 Core Clock 만 설정하고 peripheral clock 클럭은 따로 설정하지 않는다
- i2c 초기화시 400KHz 동작 클럭 설정위해  Chip_I2C_SetClockRate() 호출시
  base clock을 12MHz로 반환하며, i2c read/wriet 동작이 제대로 되지 않는다.
- spi 초기화 단계에서 16Mhz  동작 클럭 설정시, base clock은 180MHz를 반환하나 실제 동작 속도는 1Mhz 로 동작한다
- system_lpc43xx.c를 sysinit.c로 변경하여 사용한다 
- Board_SetupClocking()에서 각 개별 주변 장치 클록에 대한 기본 클록 소스를 설정한다. 

    /* Set up and initialize clocking prior to call to main */
    void Board_SetupClocking(void)
    {
        int i;
        
        Chip_Clock_SetBaseClock(CLK_BASE_SPIFI, CLKIN_IRC, true, false);

        /* Setup FLASH acceleration to target clock rate prior to clock switch */
        Chip_CREG_SetFlashAcceleration(MAX_CLOCK_FREQ);

        Chip_SetupCoreClock(CLKIN_CRYSTAL, MAX_CLOCK_FREQ, true);

        // Setup system base clocks and initial states. This won't enable and
        // disable individual clocks, but sets up the base clock sources for
        // each individual peripheral clock.

        for (i = 0; i < (sizeof(InitClkStates) / sizeof(InitClkStates[0])); i++) {
            Chip_Clock_SetBaseClock(InitClkStates[i].clk, InitClkStates[i].clkin,
                                    InitClkStates[i].autoblock_enab, InitClkStates[i].powerdn);
        }

        /* Reset and enable 32Khz oscillator */
        LPC_CREG->CREG0 &= ~((1 << 3) | (1 << 2));
        LPC_CREG->CREG0 |= (1 << 1) | (1 << 0);

        /* Setup a divider E for main PLL clock switch SPIFI clock to that divider.
        Divide rate is based on CPU speed and speed of SPI FLASH part. */
    #if (MAX_CLOCK_FREQ > 180000000)
        Chip_Clock_SetDivider(CLK_IDIV_E, CLKIN_MAINPLL, 5);
    #else
        Chip_Clock_SetDivider(CLK_IDIV_E, CLKIN_MAINPLL, 4);
    #endif
        Chip_Clock_SetBaseClock(CLK_BASE_SPIFI, CLKIN_IDIVE, true, false);
    }

7. ade9000 ISR에서 xTaskNotifyFromISR() 호출시 HardFault_Handler 예외 발생
- sct 파일에서 stack 영역을 4kb -> 32kb로 확장(process stack, not isr) (X)
- interrupt priority를 ethernet과 동일하게 설정(시험중)

    [Cortex-M4 우선순위 시스템]
    Cortex-M4에서 우선순위 값이 낮을수록 우선순위가 높음.
    우선순위는 NVIC에서 상위 비트만 사용(대부분 4비트).
    [FreeRTOS 규칙]
    ISR에서 FreeRTOS API를 호출하려면:
    ISR의 우선순위는 configMAX_SYSCALL_INTERRUPT_PRIORITY보다 낮아야 함.
    

    configMAX_SYSCALL_INTERRUPT_PRIORITY 	191
    // ISR에서 FreeRTOS API를 호출하려면:
    // ISR의 우선순위는 configMAX_SYSCALL_INTERRUPT_PRIORITY보다 낮아야 함.
    // 즉 191 보다 큰 192 부터 설정한다 (6 << 5): 0xC0(192)


- Cortex-M4는 두 개의 스택 포인터를 지원하며, 특정 작업에 따라 스택을 구분합니다.

    A. Main Stack Pointer (MSP)
    주로 인터럽트 및 예외 처리 시 사용됩니다.
    초기화 후 기본 스택 포인터로 설정됩니다.
    startup.s에서 스택 초기 주소와 크기를 설정합니다.
    B. Process Stack Pointer (PSP)
    태스크 기반의 애플리케이션에서 태스크 스택을 관리하기 위해 사용됩니다.
    FreeRTOS와 같은 RTOS 환경에서 태스크마다 독립적인 PSP를 사용.
